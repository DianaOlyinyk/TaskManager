Data Layer (DL)
1. Project

Представляє сутність проєкту, яка зберігається у сховищі.
Містить основні поля та колекцію пов’язаних завдань.
Має обчислюване поле прогресу, що визначає відсоток виконаних завдань.

2. Task 

Представляє завдання, яке належить конкретному проєкту.
Містить інформацію про назву, опис, пріоритет, дедлайн та статус виконання.
Має обчислюване поле, що визначає, чи є завдання простроченим.

3. ProjectType (enum)

Тип проєкту (навчальний, робочий, особистий).

4. Priority (enum)

Пріоритет завдання (низький, середній, високий).

Data Access Layer (DAL)
1. IProjectRepository

Інтерфейс, який визначає базові операції доступу до даних проєктів

2. ProjectRepository

Реалізація інтерфейсу IProjectRepository.
Відповідає за взаємодію зі сховищем.
Забезпечує збереження та читання даних.

Business Logic Layer (BL)
1. IProjectService

Інтерфейс сервісного шару для роботи з проєктами.
Описує методи отримання списку проєктів, деталей проєкту, а також операції додавання, редагування та видалення.

2. ProjectService

Реалізація IProjectService.
Виконує бізнес-логіку:
отримує дані з репозиторію
перетворює моделі у DTO
забезпечує цілісність даних

3.ITaskService

Інтерфейс для роботи із завданнями.

4. TaskService

Реалізація ITaskService.
Отримує дані про завдання через репозиторій та перетворює їх у DTO.

DTO (Data Transfer Objects)

1. ProjectListDto

Спрощене представлення проєкту для відображення у списку.

2. ProjectDetailsDto

Розширене представлення проєкту для сторінки деталей.

3. TaskListDto

Спрощене представлення завдання для відображення у списку.

4. TaskDetailsDto

Повне представлення завдання для детального перегляду.


UI Layer (WinUI)

1. ProjectsViewModel

ViewModel для сторінки списку проєктів.
Відповідає за: отримання списку проєктів із сервісу
фільтрацію, сортування
взаємодію з UI через binding

2. ProjectDetailsViewModel

ViewModel для сторінки деталей проєкту.
Відповідає за: завантаження даних про конкретний проєкт
відображення списку завдань
підготовку даних для UI (обчислювальні властивості)

3. TaskDetailsViewModel

ViewModel для сторінки деталей завдання.
Відповідає за: отримання інформації про конкретне завдання
форматування даних для відображення


Pages (View)
1. ProjectsPage

Головна сторінка застосунку.
Відображає список проєктів та забезпечує навігацію до деталей.

2. ProjectDetailsPage

Сторінка деталей проєкту.
Відображає інформацію про проєкт і список його завдань.

3. TaskDetailsPage

Сторінка деталей завдання.
Відображає повну інформацію про вибране завдання.

Infrastructure

1. App

Головний клас застосунку.
Виконує реєстрацію сервісів і ViewModel та запуск головного вікна

2. MainWindow

Головне вікно застосунку.
